2. 変数の定義

2-1. データ期間

  • 注文データは出荷日、返品データは返品登録日、入荷データは入荷日が基準
  • ある月の返品データは、その月より過去の月の注文分を含む

2-2. 商品名

  • 単価が0でない、品番と一意に一致する商品名

2-3. 商品売上

  • 単価 × 数量

2-3. 売上金・購入金額・請求金額

  • (単価 × 数量) + 商品消費税 + 送料 + 手数料 + 調整金 + サービス消費税
  • 同一注文番号に同じ品番のデータが含まれる場合、数量は合計、単価は平均を取ったものになる
  • 商品名でフィルタリングしたときは、商品価格+その商品を含む注文のサービス価格

2-4. エリア

  • 送付先都道府県
  • 顧客の住所は引っ越しで変わるなどの理由で送付先を利用

2-5. 注文数

  • ユニークな注文番号の数

2-6. 顧客数

  • ユニークな顧客番号の数

2-7. 累計購入金額

  • 期間内で、顧客一人あたりの平均で、累計の購入金額
  • 全データ表示状態ではLTVと等しい

2-8. ABC分析

  • 商品をAランク(売れ筋)、Bランク(中堅)、Cランク(死に筋)の3つに分類する
  • 商品ごとに、商品売上を降順ソートし、累積売り上げを求めたときの、70%までがAランク、95%までがBランク、残りがCランクとなる
  • セット商品など、商品売上が正しく入力されていない商品では機能しない

2-9. リピート数

  • 同じ顧客番号を持つ別注文番号の数
  • 購入回数nのリピート数がR(n-1)となる

2-10. 継続率

  • n/(n-1)で表される、前回購入した人のうちどれだけが次回も購入したかを表す値
  • 実際に離脱したり、最近の購入者が増えると継続率の数値が下がる

2-11. 購入日

  • 元データが持っている粒度で、秒未満切り捨て

2-12. 返品数・返品件数

  • 注文日が返品データと同期間に存在する、ユニークな返品IDの数
  • 最も古い返品申請日より注文日が前のデータと、未来の返品データは含まれない
  • 同一注文を複数に分けて返品した場合、分割された返品数

2-13. 返品率

  • 返品IDに紐づいた注文数(≠返品件数) / 注文数
  • 同一注文を複数に分けて返品した場合、分子は重複しない
  • ある月の注文数に対する返品数は、今後返品される分を含まないため、直近のデータは低く出る

2-14. 返品金額

  • 注文日が返品データと同期間に存在する、返品IDを持つ注文番号の売上金
  • 最も古い返品申請日より注文日が前のデータと、未来の返品データは含まれない
  • 商品名でフィルタリングしたときは、商品価格+その商品を含む注文のサービス価格
  • 注文の一部しか返品されなかった場合も、返品額は注文の全額
  • 同一注文を複数に分けて返品した場合は、重複しない

2-15. 返品コスト

  • 返品着払い送料

2-16. 在庫増減

  • 入荷数 - 出荷数 + 返品数

3. 加工処理

  • 正規化処理
    • 入力データのデータ型に従い、keyのないデータを破棄する
    • strのデータ欠損は空白文字列で埋める
    • datetimeのデータは標準日付フォーマット[YYYY-MM-DD HH:mm:ss]に変換する(時刻欠落の場合は0埋め)
  • クレンジング
    • 出荷データ
      • 請求金額0のデータを除外する
      • 商品単価×数量を商品代金、送料+手数料をサービス料とする
      • 消費税を商品代金とサービス料に金額比で分割し、商品消費税とサービス消費税とする(端数切捨て)
      • 請求金額から商品代金、商品消費税、サービス料、サービス消費税を差し引いた額を調整金とする
      • 注文ごとに固有の情報を注文明細とする
        • {∄!注文番号・出荷日・送付先・配送方法・支払方法・顧客番号・送料・手数料・調整金・サービス料・サービス消費税}
      • 単価と数量が0のデータを除外する
        • 商品フィルターをかけたときに、送料等のみの表示になるなど誤解を招くため
      • 同一注文内の同品番を持つデータの平均単価と合計数量を算出して1つに統合する
      • 注文番号+品番に固有の情報を商品出荷テーブルとする
        • {注文番号・品番・単価・数量・商品代金・商品消費税}
      • 同一年月のデータと結合する
        • 注文番号、または注文番号+品番をハッシュ化し、同一のindexを持つデータは新しいデータで上書きする
    • 返品データ
      • 返品着払い送料を返品コストとする
      • 注文番号で注文データと紐づけ、商品の情報や注文の情報は注文データから取得する
        • 注文と異なる商品が返品されたとき、商品ごとの正しい返品率が反映されなくなるため
      • 注文データに含まれない、返品IDに固有の情報を返品マスタとする
        • {∄!返品ID・注文番号・返品理由・返品コスト}
      • 注文データに含まれない、同じ返品IDの詳細情報を返品テーブルとする
        • {返品ID・返品アイテム(≠注文商品)・数量}
      • 同一返品内の同品番を持つデータの合計数量を算出して1つに統合する
      • 返品IDに紐づいた注文番号を持つ注文はすべて返品されたとする
        • 注文の一部が返品された場合も返金額は注文全額
        • 同一注文を複数に分けて返品した場合は重複しない
        • 商品フィルタは返金額のうちその商品が占める金額となる
      • 同一年月のデータと結合する
        *返品番号、または返品番号+品番をハッシュ化し、同一のindexを持つデータは新しいデータで上書きする
